# FileSort
 Локальный интеллектуальный органайзер файлов с семантической классификацией и статическим анализом безопасности

 Задача
Пользователи Windows ежедневно скачивают десятки файлов: документы, архивы, изображения, скрипты. Папка «Загрузки» и рабочий стол превращаются в свалку, где нужный файл приходится искать минутами. Существующие инструменты автоматизации либо работают по жёстким правилам (расширение → папка), не понимая смысла содержимого, либо требуют отправки данных в облако, что неприемлемо для конфиденциальной информации.

Для кого
Обычные пользователи, уставшие от цифрового хаоса.

Специалисты, работающие с разными типами файлов (документы, код, медиа) — разработчики, дизайнеры, аналитики.

Все, кто заботится о приватности и не хочет загружать свои файлы на внешние серверы.

Какая боль сейчас
Ручная сортировка отнимает время. Даже 5–10 минут в день превращаются в часы в месяц.

Правила «если-то» не работают для семантики. Файлы отчет_2024.docx и `счет_ivanov.pdf» относятся к работе, но не имеют общих ключевых слов в названии.

Опасные скрипты маскируются под безобидные .txt или .doc. Пользователь может случайно запустить вредоносный код.

Облачные сервисы анализируют содержимое, что нарушает конфиденциальность корпоративных или личных данных.

Что делает PoC на демо
Принимает входящий файл (любого типа) и бесшумно выполняет четыре шага:

Мониторинг в реальном времени.
Отслеживание папки (например, «Загрузки») через FileSystemWatcher. Новый файл мгновенно поступает в обработку.

Извлечение текста и метаданных.
Для документов (DOCX, PDF, TXT) – текст; для изображений – EXIF; для архивов – список файлов; для скриптов – исходный код.

Локальный семантический анализ через LLM (Llama 3.2 3B).
Модель определяет категорию файла (рабочий документ, личное фото, счёт, скрипт и т.д.) и предлагает имя целевой папки на языке пользователя. Одновременно выполняется статический анализ безопасности для скриптов (поиск подозрительных вызовов, обфускации).

Перемещение с подтверждением для скриптов.
Система создаёт (при необходимости) папку с осмысленным именем и перемещает файл. Для скриптов перед перемещением выводится предупреждение о рисках — пользователь сам решает, разрешить действие или нет.

Data Flow:

text
Файл ──▶ FileSystemWatcher ──▶ Извлечение текста ──▶ Локальная LLM (Llama 3.2) ──▶ Security Scan ──▶ Решение о папке ──▶ Перемещение (с подтверждением для скриптов)
Что НЕ делает PoC (out-of-scope)
Не обрабатывает файлы размером более 1 ГБ (ограничение производительности локальной модели).

Не запускает скрипты — только статический анализ.

Не имеет мобильной версии — только Windows Desktop MVP.

Не интегрируется с облачными хранилищами (Google Drive и т.п.) — только локальная файловая система.

Не выполняет автоматических действий без подтверждения пользователя для потенциально опасных файлов.

Стек
Компонент	Технология
Мониторинг файлов	System.IO.FileSystemWatcher (через Python.NET или watchdog)
Извлечение текста	python-docx, PyPDF2, zipfile, PIL для изображений
Локальная LLM	Llama 3.2 3B через Ollama или Nexa SDK
Классификация	Промптинг + HiClass (иерархическая классификация)
Статический анализ	Regex-правила, AST (для Python-скриптов), эвристики
Хранение логов	SQLite / JSONL (локально)
Интерфейс	Tray-иконка (Windows) + лог-окно (PyQt / tkinter) для настроек
Оркестрация	Python 3.11, асинхронная обработка очереди
Возможные расширения
Поддержка macOS/Linux через аналоги inotify / FSEvents.

Обучение собственной лёгкой модели для классификации без LLM (эмбеддинги + классификатор).

Интеграция с антивирусными сканерами (ClamAV) для углублённой проверки.

Веб-интерфейс для просмотра истории и статистики.

Правила пользователя (гибрид: семантика + жёсткие условия).

Автоматическое переименование файлов по шаблону на основе содержимого.

Безопасность и приватность (из Governance)
Весь анализ выполняется строго локально, данные не покидают устройство.

Для скриптов применяется статический анализ без запуска.

Любое действие со скриптами требует явного подтверждения пользователя.

Ведётся локальный журнал всех операций для аудита.
