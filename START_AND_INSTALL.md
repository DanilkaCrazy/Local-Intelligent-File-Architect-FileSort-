# 🚀 FileSort Organizer - Руководство по установке и запуску

**FileSort Organizer** — это умное десктопное приложение для автоматической сортировки файлов. Оно использует локальную нейросеть для анализа названий файлов и их безопасного распределения по категориям, а также включает встроенную проверку безопасности для скриптов.

**Автор:** Вишняков Данил

---

## 🛠 Шаг 1. Подготовка нейросети (Обязательно)

Так как программа работает **полностью локально** и не отправляет ваши файлы в интернет, ей требуется локальная нейросеть для анализа.

1. Скачайте и установите **Ollama** с официального сайта: [https://ollama.com/](https://ollama.com/)
2. После установки откройте терминал (или командную строку Windows) и введите команду для загрузки нужной модели:
   ```bash
   ollama run llama3.2
   ```
   *(Дождитесь окончания загрузки. Модель весит около 2-3 ГБ. После загрузки можете закрыть этот терминал, главное чтобы значок Ollama висел в трее рядом с часами).*

---

## 📦 Шаг 2. Установка программы

У вас есть два варианта: использовать готовый установщик или собрать программу самостоятельно из исходного кода.

### Вариант А: Использование готового файла (Для обычных пользователей)
Если вы уже скомпилировали проект, готовые файлы находятся в папке `src-tauri/target/release/`:
- **Portable версия (без установки):** Просто запустите `filesort-organizer.exe`.
- **Установщик:** Зайдите в папку `bundle/nsis/` и запустите `FileSort Organizer_0.1.0_x64_setup.exe`. Следуйте стандартным шагам установки Windows.

### Вариант Б: Сборка из исходного кода (Для разработчиков)
Если вы только скачали исходный код (ZIP-архив) и хотите собрать `.exe` файл:

1. Убедитесь, что у вас установлены:
   - **Node.js** (https://nodejs.org/)
   - **Rust** (https://rustup.rs/)
   - **C++ Build Tools** (требуются для компиляции Rust под Windows)
2. Откройте терминал в папке проекта и установите зависимости:
   ```bash
   npm install
   ```
3. Запустите процесс сборки:
   ```bash
   npm run tauri build
   ```
4. После успешного завершения заберите готовый `.exe` файл по путям, указанным в Варианте А.

## 🖥 Шаг 3. Как пользоваться программой

1. **Запустите Ollama:** Убедитесь, что Ollama работает в фоновом режиме (значок ламы в системном трее).
2. **Откройте FileSort Organizer.**
3. **Выберите исходную папку (Source):**
   - Нажмите кнопку `Select Source Folder` в правом верхнем углу.
   - Выберите папку, в которой сейчас свалка файлов (например, папку "Загрузки").
4. **Выберите целевую папку (Target):**
   - Нажмите кнопку `Select Target Folder`.
   - Выберите пустую папку, куда программа должна аккуратно сложить файлы.
5. **Запустите сортировку:**
   - Нажмите синюю кнопку `Scan & Organize`.
   - Программа начнет сканировать исходную папку. Каждый файл будет отправлен в локальную нейросеть для определения категории (Document, Image, Archive и т.д.).
   - Файлы будут физически перемещены в целевую папку и разложены по подпапкам.
6. **Проверка безопасности:**
   - Если программа наткнется на исполняемый скрипт (например, `.py`, `.ps1`, `.bat`), она **приостановит работу** и покажет красное окно с предупреждением.
   - Вы должны вручную решить: разрешить перемещение этого скрипта (`Allow Move`) или пропустить его (`Block & Skip`).

## ❓ Решение частых проблем

- **Файлы попадают в категорию "Other" или программа зависает на этапе "Analyzing..."**
  *Причина:* Ollama не запущена или не скачана модель `llama3.2`.
  *Решение:* Убедитесь, что Ollama работает. Откройте терминал и проверьте команду `ollama list`.

- **Ошибка при сборке (error RC2176 : old DIB in icon.ico)**
  *Причина:* Конфликт форматов иконок в Windows.
  *Решение:* Откройте файл `src-tauri/tauri.conf.json`, найдите блок `"bundle" -> "icon"` и оставьте массив пустым: `"icon": []`. Затем снова запустите `npm run tauri build`.

- **Не перемещаются файлы с диска C: на диск D:**
  *Решение:* Эта проблема уже решена в коде (добавлен fallback на копирование+удаление при ошибке cross-device link). Если ошибка всё же возникает, запустите программу от имени Администратора.
