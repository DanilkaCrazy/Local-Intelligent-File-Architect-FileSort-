## 2. Политика логов

### Что логируется

| Событие                         | Данные в логе                                                                                          | Хранение                         |
| ------------------------------- | ------------------------------------------------------------------------------------------------------ | -------------------------------- |
| Запуск приложения               | timestamp, версия приложения, идентификатор сессии (локальный)                                         | Локальный файл `logs/app.log`    |
| Аутентификация пользователя     | timestamp, метод входа (Google/email), успех/неудача, IP (анонимизированный)                           | `logs/auth.log`                  |
| Обнаружение нового файла        | timestamp, исходный путь, размер, хеш (SHA-256)                                                        | `logs/watcher.log`               |
| Извлечение текста/метаданных    | timestamp, тип файла, успех/ошибка, длина извлечённого текста                                          | `logs/extractor.log`             |
| Промпт к LLM                    | timestamp, модель, количество токенов (вход/выход), длительность                                      | `logs/llm.log` (без полного текста) |
| Ответ LLM                       | timestamp, предложенная категория, уверенность, предлагаемое имя папки, флаги безопасности (для скриптов) | `logs/llm.log`                   |
| Решение о перемещении           | timestamp, исходный путь, целевой путь, категория, источник решения (LLM/правила), режим (dry run/реальный) | `logs/decision.log`              |
| Действие с файлом               | timestamp, операция (move/copy/delete), исходный путь, целевой путь, успех/ошибка                     | `logs/fs.log`                    |
| Подтверждение пользователя      | timestamp, действие, результат подтверждения (разрешено/отказано)                                     | `logs/user_interaction.log`      |
| Ошибка компонента               | timestamp, модуль, код ошибки, описание                                                                | `logs/error.log`                 |

### Что НЕ логируется

- Полное содержимое файлов (текст, бинарные данные).
- Пароли, токены доступа (кроме факта успешной аутентификации).
- API-ключи (если используются для внешних сервисов — в MVP не используются).
- Персональные данные пользователя (email логируется только факт входа, но не сам email в открытом виде? В целях аудита можно хранить хеш email, но лучше минимизировать).

### Формат логов

Структурированные логи в формате JSON, одна строка — одно событие. Пример:

```json
{"timestamp":"2025-03-06T10:15:30Z","level":"INFO","module":"watcher","event":"file_detected","path":"C:\\Users\\User\\Downloads\\report.pdf","size":204800,"hash":"sha256:..."}
